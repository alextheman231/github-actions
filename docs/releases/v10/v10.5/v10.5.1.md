## Description of Changes

<!-- user-editable-section-start -->
- Use the new `safe-npm-dependency-global-install` composite action from the new `typescript-actions` repository to safely global install `alex-c-line` into workflows.
    - This is better than directly global-installing `alex-c-line` in a workflow as this will give us more control over exactly what version we use.
    - For my use cases, it will use whatever version is in `package.json` if it is present, otherwise it will error.
    - There are options to override this using a direct `version_range` input along with a `strict_version_resolution`, but you can see more information about that composite action in general in [release note v1.0.0 of `typescript-actions`](https://github.com/alextheman231/typescript-actions/releases/tag/v1.0.0)
