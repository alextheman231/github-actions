This is a new patch release of the `github-actions` package. It includes small, non-breaking changes and should require no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Use the new `safe-npm-dependency-global-install` composite action from the new `typescript-actions` repository to safely global install `alex-c-line` into workflows.
    - This is better than directly global-installing `alex-c-line` in a workflow as this will give us more control over exactly what version we use in our workflows.
    - For my use cases, it will use whatever version is in `package.json` if it is present, otherwise it will error if `strict_version_resolution` is true.
    - There are options to override this using a direct `version_range` input along with a `strict_version_resolution`, but you can see more information about that composite action in general in [release note v1.0.0 of `typescript-actions`](https://github.com/alextheman231/typescript-actions/releases/tag/v1.0.0)
- Refactor `actions-ci` to use the official Zizmor composite action rather than directly install from Homebrew/pip
    - This allows us to better control it through SHA pinning/weekly updates, rather than always install Zizmor at latest in the workflow and potentially encounter breaking changes despite us not explicitly doing anything to trigger it in our own code.
