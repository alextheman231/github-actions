This is a new patch release of the `github-actions` package. It includes small, non-breaking changes and should require no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Explicitly list the generated app token's permissions per usage
    - A recent Zizmor rule has been introduced to enforce that we explicitly pass in the exact permissions for the app token we are trying to use.
    - This makes the usage more secure, ensuring that the generated token itself contains only the permissions we allow it to have rather than all the possible permissions it can have at installation.

## Additional Notes

- Hope this doesn't break anything!
- It affects so many of our reusable workflows, so there is actually a bit of a chance this may break them.
- If that happens, the fix should generally just be to add another permission, but for now I've pretty much just used the same permissions that the workflow itself has, so it should be fine.
